## Maintainer Expectations

- Keep `main` green.
- Keep public claims scoped to working, reviewed surfaces.
- Require tests for behavior changes.
- Treat external package metadata as untrusted input.
- Keep API compatibility notes in release entries.
- Keep public repository history professional.
- Use verified commit signing for maintainer-led repository changes.

## Access Review

Repository access should be reviewed before every major release and after any security incident.

Access should be removed when a person or service no longer needs write access.
