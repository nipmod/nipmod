# Decision Records

Nipmod uses decision records for durable product and architecture choices.

Use a decision record when a change affects:

- public API shape
- source resolver policy
- trust scoring
- archive persistence
- security boundary
- pricing or payment boundary
- public product positioning
- governance

## Template

```md
# 0000 - Title

Status: proposed | accepted | superseded
Date: YYYY-MM-DD

## Context

## Decision

## Alternatives

## Impact

## Security

## Rollback
```
