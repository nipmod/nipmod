# Research Cadence

Nipmod strategy should be updated through evidence, not mood.

## Weekly Checks

Run a weekly market pass over:

- package registry changes
- MCP registry changes
- agent platform package workflows
- open source supply-chain incidents
- security product launches
- package search and discovery products
- developer reactions on GitHub, X, Discord, Telegram and forums

## Monthly Decisions

Once per month, update:

- source priority list
- competitor table
- pricing hypothesis
- trust scoring gaps
- docs and website claim boundaries
- active outreach targets

## Decision Memo Template

Use this before major product or positioning changes:

```text
Decision:

Context:

Options:

Recommended option:

Why this improves Nipmod:

Risks:

What would prove this wrong:

What changes in code/docs/product:

Public claim allowed:

Public claim not allowed:
```

## Claim Rules

Allowed:

- Nipmod provides one package intelligence API for agents.
- Nipmod resolves public package sources and returns trust context and safe install plans.
- Hosted API calls are read-only.
- Confirmed package records can be stored in the Nipmod archive.
- Gitlawb is a supported source.

Not allowed without proof:

- official integration with a platform
- endorsement by a source network or agent host
- legal ownership of external packages
- guaranteed malware prevention
- guaranteed investment outcome
- token price or market promises
