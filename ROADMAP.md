# Roadmap

Nipmod is moving toward one package API for agents.

## Now

- Hosted package search across supported public sources.
- Package inspect records with source links, metadata and trust signals.
- Safe install plans that agents can show before workspace writes.
- Durable archive preparation for confirmed useful package records.
- Local CLI and MCP server for controlled workspace workflows.

## Next

- API keys for higher rate limits.
- Stronger trust scoring with vulnerability, license and maintainer signals.
- Durable archive writes with abuse controls, quotas and audit logs.
- TypeScript SDK and Python SDK.
- More source resolvers where public APIs and terms allow it.
- Versioned OpenAPI documentation.

## Later

- Owner claim flow for external package records.
- Premium package access once free public usage proves demand.
- x402 compatible payment flows for paid API or premium package access.
- Public enterprise controls for audit export, policy packs and allowlists.

## Rules

- Free public beta stays rate limited.
- External package owners keep ownership.
- Nipmod does not rehost third party packages without permission or a clear license path.
- New source claims require tests and public docs before marketing claims.
